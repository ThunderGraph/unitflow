Implmenation plans must include
- table of contents
- high level discussion of methodology and methods
- very limited code details, only where truly essential for explaining methodology
- mermaid diagrams in dark mode when need to explain architecture
- file tree architecture
- references if external sources are used.
- phased gate go/nogo phases
- a test plan that describes what should be tested with both unit tests and integration test. A
  description is a description, not code!

ThunderGraph design philosophy is as such:
- Simpler more elegant easier to understand architectures are best as long as they meet
  requirements
- Follow SOLID principles when coding
- Functions should not be too long and should only do one thing.

The author should instill the design philosphy into the plan.
