# UnitFlow Release Notes

## v0.1.0: Initial Release

**Stop converting. Start computing.** 

We are excited to announce the initial release of **UnitFlow** — an elegant, correct, and composable engineering math substrate for Python. Designed as the computational foundation for [ThunderGraph](https://www.thundergraph.ai/), UnitFlow brings rigorous dimensional reasoning and symbolic constraint authoring to modern engineering and MBSE (Model-Based Systems Engineering) workflows.

### 🌟 Key Features in this Release

#### 1. Core Semantic Algebra
* **Immutable Units & Dimensions:** Build units via exact rational arithmetic (fractions) and explicit $\pi$ powers to prevent floating-point drift.
* **Semantic Equality:** 1 meter and 100 centimeters are semantically equal. The library tracks dimension vectors natively.
* **Quantity Kinds:** Disambiguate units that share physical dimensions but have different engineering meanings (e.g., Torque vs. Energy).

#### 2. First-Class NumPy Integration
* **Array-backed Quantities:** Wrap NumPy arrays in `Quantity` objects without monkey-patching.
* **UFunc Dispatch:** Seamlessly supports `np.add`, `np.multiply`, `np.square`, `np.sum`, `np.mean`, etc., preserving or converting units on the fly.
* **Transcendental Safety:** Functions like `np.sin`, `np.cos`, and `np.exp` explicitly require dimensionless array inputs, catching math errors before they propagate.

#### 3. Symbolic Expression Layer for MBSE
* Define symbolic `Symbol`, `Expr`, and `Constraint` objects to build executable equations and inequalities.
* Compose inequalities using standard python logic (`&`, `|`, `~`).
* *Note: This layer is currently structural and designed for AST traversal/serialization, setting the stage for future solver integration.*

#### 4. Extensibility & Catalogs
* **Curated Catalogs:** Ships with robust `si` and `mech` unit catalogs.
* **Programmatic Prefix Generation:** Automatically generate all SI orders of magnitude (milli, micro, kilo, mega, etc.) for any base unit using `generate_prefixes`.
* **Custom Namespaces:** Easily define domain-specific user units without touching library internals.

#### 5. Serialization
* **JSON-safe:** All core semantic objects and constraint trees serialize natively to plain dictionaries for easy distributed execution and digital twin persistence. *(Note: Array magnitudes are not currently serializable).*

### 🛠️ Example Usage

```python
import numpy as np
from unitflow import si, kW, rpm, rad, symbol

# Exact pi-bearing conversions
angular = (3000 * rpm).to(rad / si.s)  # 314.159... rad/s

# NumPy Integration
masses = np.array([1.0, 2.0, 5.0]) * si.kg
accels = np.array([9.81, 9.81, 9.81]) * si.m / si.s**2
forces = masses * accels
print(np.sum(forces))  # 78.48 kg*m/s^2

# Symbolic Models
shaft_speed = symbol("shaft_speed", unit=rpm)
speed_bounds = (0 * rpm <= shaft_speed) & (shaft_speed <= 6000 * rpm)
```
